## 6. ACCESSIBILITY REQUIREMENTS

### 6.1 ARIA Labels & Attributes

- All icon buttons: `aria-label="Action description"`
- Feature cards: `role="article"`, `aria-describedby="feature-description-{id}"`
- Modals: `role="dialog"`, `aria-modal="true"`, `aria-labelledby="dialog-title"`
- Tab panels: `role="tabpanel"`, `aria-labelledby="tab-{id}"`
- Live regions: `aria-live="polite"` for updates, `aria-live="assertive"` for critical events

### 6.2 Keyboard Navigation

- Tab order follows visual hierarchy
- Focus indicators visible (2px outline, primary color)
- Skip link to main content
- All interactive elements accessible via keyboard
- No keyboard traps

### 6.3 Screen Reader Support

- Descriptive image alt text (or aria-hidden="true" if decorative)
- Link text meaningful (not "Click here")
- Form labels associated with inputs
- Error messages associated with fields
